## 🚀 Новый gRPC метод

```go
func (s *API) NewMethod(ctx context.Context, req *pb.Request) (*pb.Response, error) {
    if err := s.validateServer(); err != nil {
        return nil, err
    }
    if err := validateRequiredFields(map[string]string{"field": req.GetField()}); err != nil {
        return nil, err
    }
    ctx, cancel, err := withTimeout(ctx, 5*time.Second)
    if err != nil {
        return nil, err
    }
    defer cancel()
    
    result, err := s.service.DoSomething(ctx, req.GetField())
    if err != nil {
        s.log.Error("failed", slog.String("error", err.Error()))
        return nil, status.Error(codes.Internal, "operation failed")
    }
    return &pb.Response{Result: result}, nil
}
```

**Interceptor уже проверяет:** `ctx != nil`, `req != nil`

## 📁 Добавление нового метода

### 1. Proto wrapper (КРИТИЧНО!)
```typescript
// shared/api/generated/user/user_pb_wrapper.ts
if (!proto.NewMethodRequest) {
  throw new Error('Proto class NewMethodRequest not found');

  throw new Error('Proto class NewMethodResponse not found');
}
```

### 2. gRPC Client
```typescript
// shared/lib/grpc-slice/grpc-client.ts
const METHODS = { NEW_METHOD: 'NewMethod' };
const ALLOWED_RESPONSE_CLASSES = new Set(['NewMethodResponse']);
// В grpcClient:
newMethod: createUserMethod(METHODS.NEW_METHOD, 'NewMethodResponse'),
```

## 🎯 Частые ошибки

| Симптом | Причина |
|---------|---------|
| `failed to unmarshal` | **Stub файлы вместо сгенерированных proto!** |
| Метод молча не работает | Забыл user_pb_wrapper.ts |
| Error deserializing response | Забыл ALLOWED_RESPONSE_CLASSES |
| Unauthenticated 401 / 0 bytes | Забыл publicMethods (backend) |
| Запросы висят | Нет timeout |
| Import error | Прямой импорт _pb (нужен wrapper) |

### ⚠️ Proto stub vs generated (КРИТИЧНО!)
```bash
# Проверить что proto сгенерированы правильно:
head -5 internal/grpc/gen/user/user.pb.go
# Должно быть: // Code generated by protoc-gen-go. DO NOT EDIT.

# Если нет — перегенерировать:
cd contract && task generate
```
