# Commit Checklist - Чеклист перед коммитом

> Этот файл - напоминание для Kiro AI перед каждым коммитом

## ⚠️ КРИТИЧНО - Проверь ПЕРЕД коммитом:

### 1. Подтверждение пользователя
- [ ] Пользователь явно подтвердил, что готов к коммиту
- [ ] Все изменения проверены и одобрены

### 2. Git команды
- [ ] **ВСЕГДА используй `git add -A`** (НЕ `git add <file>`)
- [ ] Используй правильный формат коммита: `feat:`, `fix:`, `docs:`, etc.
- [ ] Проверь, что находишься в правильной директории

### 3. Репозитории
<!-- TODO: Настроить под ваш проект -->
- [ ] Frontend: `frontend/`
- [ ] Backend: `backend/`
- [ ] Contract: `contract/`
- [ ] Документация: `.kiro/`

### 4. Последовательность команд

```bash
# ПРАВИЛЬНО ✅ - Одна команда (рекомендуется)
git add -A && git commit -m "feat: описание" && git push

# ПРАВИЛЬНО ✅ - Три команды (если нужно проверить между шагами)
git add -A
git commit -m "feat: описание"
git push

# НЕПРАВИЛЬНО ❌
git add file1.go file2.go  # НЕ ДЕЛАЙ ТАК!
git commit -m "feat: описание"
git push
```

**Почему одна команда лучше:**
- ✅ Быстрее - всё за один раз
- ✅ Меньше ошибок - не забудешь push
- ✅ Атомарность - либо всё, либо ничего

## 🎯 Почему `git add -A`?

1. **Захватывает ВСЕ изменения** (новые, изменённые, удалённые файлы)
2. **Не пропускает файлы** - гарантия полноты коммита
3. **Быстрее** - одна команда вместо нескольких
4. **Меньше ошибок** - не нужно перечислять файлы вручную

**⚠️ Monorepo:** Проект содержит backend, frontend, contract в одном репо. `git add -A` из корня захватит ВСЁ. Если нужен атомарный коммит только для одной части — убедись что другие изменения готовы к коммиту или используй `git stash` для незавершённых.

## 📝 Примеры правильных коммитов

```bash
# Backend изменения (одна команда)
cd backend
git add -A && git commit -m "feat: добавить SQL injection protection" && git push

# Документация (одна команда)
cd .kiro
git add -A && git commit -m "docs: обновить tasks.md - Task 1.1 завершён" && git push

# Frontend изменения (одна команда)
cd frontend
git add -A && git commit -m "fix: исправить валидацию формы регистрации" && git push
```

## 🚨 Если забыл `git add -A`

Если уже сделал коммит с `git add <file>`:
1. Проверь `git status` - есть ли незакоммиченные файлы
2. Если есть - сделай дополнительный коммит с `git add -A`
3. В следующий раз используй этот чеклист!

## 💡 Как запомнить

**Мантра:** "git add -A && commit && push - ОДНОЙ КОМАНДОЙ!"

Перед каждым коммитом:
1. Открой этот файл
2. Проверь чеклист
3. Используй `git add -A && git commit -m "..." && git push`
4. Profit! 🎉
