## Бизнес-сущности

### User (Пользователь)
Основная модель. Содержит email, телефон, имя, роль, статус. Связан с UserProfile и RefreshToken.

### UserProfile (Профиль)
Расширяемый профиль пользователя: аватар, био. Связь: `user_profiles.user_id → users.id`.

### RefreshToken (Токен обновления)
Хранит SHA256 хеш refresh токена. Привязан к пользователю, имеет срок действия и флаг отзыва.

### RegistrationProgress (Прогресс регистрации)
Multi-step регистрация: хранит телефон, текущий шаг, JSON данные шагов. TTL 24 часа.

## Статусы

### active (Активный)
Работающий пользователь, доступен для входа и использования системы.

### inactive (Неактивный)
Пользователь деактивирован. Не может войти в систему.

### blocked (Заблокирован)
Заблокирован за нарушения или администратором. Не может войти.

### deleted (Удалён)
Soft-deleted пользователь. Запись остаётся в БД, но недоступна.

## Технические термины

### gRPC-Web
Протокол для вызова backend API из браузера через HTTP/1.1.

### Proto wrapper
Обёртка над сгенерированными proto файлами для совместимости с ES modules.

### Serverless Container
Контейнер в YC, который запускается по запросу и масштабируется автоматически.
